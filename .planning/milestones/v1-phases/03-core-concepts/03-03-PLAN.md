6. "## 개념: 메모리 플러시 타이밍" — RESEARCH.md "Memory Flush": 변경은 디스크에 즉시 저장되지만 시스템 프롬프트에는 **다음 세션**부터 반영(스냅샷이 세션 시작 시 동결). 이것이 독자가 가장 자주 혼동하는 지점임을 명시.

7. "## 개념: 세션 검색 (session_search 툴)" — RESEARCH.md "Session Search":
   - **session_search는 에이전트 내부 툴**이며 슬래시 커맨드가 아님. 트리거하려면 자연어로 질문("지난주에 X에 대해 얘기한 내용이 뭐였지?")하면 에이전트가 자율적으로 호출.
   - `~/.hermes/state.db`를 SQLite FTS5로 쿼리. 세 호출 패턴(discovery/scroll/browse). raw 메시지 반환(LLM 요약/절단 없음). ~20ms FTS5 쿼리, ~1ms scroll. 설정 불필요, 항상 사용 가능.
   - RESEARCH.md "FTS5 tables" 표(`messages_fts` 표준, `messages_fts_trigram` 트라이그램/CJK). **한국어 독자 callout**: messages_fts_trigram이 v10+에서 CJK/부분 문자열 검색을 위해 추가되어 대화 이력의 한국어 콘텐츠가 완전히 검색 가능함.
   - **주의**: RESEARCH.md Open Question #7대로 "4,500x faster" 수치는 인용하지 말 것 — "~20ms FTS5 쿼리"만 사용.
   - 고급 독자용 직접 DB 쿼리 코드블록: RESEARCH.md "Direct DB query" sqlite3 예시 그대로. 첫 줄 `# 검증: SQLite 로컬 실행 필요, 2026-06-09`. 그 아래 `> 검증 필요:` callout으로 "스키마는 공식 session-storage 문서 기준이나 로컬 state.db에서 실행 확인 후 게재 권장" 명시.
