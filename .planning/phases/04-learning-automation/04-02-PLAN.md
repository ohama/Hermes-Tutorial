1. 제목: `# 학습 루프`
   상단 "이 챕터를 시작하기 전에" callout: 선행 [Ch.9 스킬 시스템](../09-skills/index.md)(스킬 생성/개선이 핵심).

2. "## 개요" — Do→Learn→Improve는 Hermes의 flagship 기능. 작업을 수행하면 에이전트가 워크플로우를 스킬로 저장 제안하고, 후속 작업에 자동 주입되며, 백그라운드 Curator가 주기적으로 품질을 검토·개선한다.

3. "## 개념: Do→Learn→Improve 사이클" — RESEARCH.md의 ASCII 다이어그램(Do → Learn → Improve → Curator)을 삽입하되, "Do" 박스에 "복잡한 다단계 작업 수행"으로 적고 "도구 5번 이상 호출" 같은 고정 트리거를 사실로 단정하지 말 것. 다이어그램은 설명용 텍스트 블록(명령 아님, `# 검증:` 불필요).

4. "## 개념: 1단계 — 스킬 자동 생성과 제안(Learn)" — 에이전트가 복잡한 작업을 성공적으로 완료하면 그 접근법을 스킬로 저장할지 **제안**하고, 사용자가 수락하면 `~/.hermes/skills/<category>/<name>/SKILL.md`에 기록된다. 정확한 트리거 조건과 제안 메시지 문구는 검증되지 않았으므로 다음 헤지를 넣을 것:
   ```
   > 검증 필요: 스킬 저장 제안의 정확한 발동 조건과 메시지 문구, 수락 방법은
   > 환경/버전에 따라 다릅니다. 아래 실습에서 로컬 실행 후 실제 출력을 캡처하십시오.
   ```

5. "## 개념: 2단계 — 사용 중 자가 개선(Improve)" — 스킬이 후속 작업에서 트리거되면 에이전트가 전체 내용을 컨텍스트에 로드해 절차를 실행하고, 새 엣지 케이스/더 나은 접근을 발견하면 스킬을 `patch`할 수 있다. patch 횟수는 메타데이터로 추적된다(`patch_count`).

6. "## 실습: 스킬이 만들어지는 과정 관찰하기" — RESEARCH.md "Observation method"를 단계로:
   1) 다단계 작업 요청 예: `hermes chat -q "이 프로젝트의 모든 Python 파일에서 미사용 import를 찾아 리포트를 만들어줘"` (코드블록 첫 줄 `# 검증: hermes rolling, 2026-06-10`).
   2) 에이전트가 여러 툴(read_file/terminal/write_file 등)을 실행.
   3) 완료 후 스킬 저장을 제안 → 수락.
   4) 제안 메시지·수락 결과는 LOW이므로 별도 코드블록에 `# [로컬 실행 후 캡처 필요 — 스킬 저장 제안 메시지 및 스킬 파일 생성 확인]` + `> 검증 필요:` callout.
   5) 생성 확인 코드블록(첫 줄 `# 검증: hermes rolling, 2026-06-10`): `hermes skills list` 그리고 `ls ~/.hermes/skills/`.

7. "## 실습: 자가 개선 흔적 확인하기" — RESEARCH.md대로 `.usage.json`로 use_count/patch_count 확인. 단, 정확한 위치/형식은 미검증이므로 코드블록(첫 줄 `# 검증: hermes rolling, 2026-06-10`)에 `cat ~/.hermes/skills/.usage.json` 시도를 적고 바로 아래 헤지:
   ```
   > 검증 필요: `.usage.json`의 정확한 위치와 형식은 로컬 실행 후 확인하십시오.
   > 파일이 없으면 `ls -a ~/.hermes/skills/`로 숨김 파일을 먼저 확인하십시오.
   ```

8. "## 개념: 3단계 — Curator 백그라운드 품질 검토" — RESEARCH.md "Step 3" 그대로. **정확성 핵심:** Curator는 작업 횟수(예: "15작업마다", "5회 툴 호출마다")가 아닌 **비활동 기반 트리거**로 실행됨 — `min_idle_hours`(기본 2시간) 경과 AND `interval_hours`(기본 168시간=7일) 경과. 신규 설치는 첫 인터벌만큼 첫 실행이 지연되어 첫 세션에서는 보이지 않는다. 백그라운드 AIAgent fork로 격리 실행되어 대화에 영향 없음.
   설정 YAML 코드블록(첫 줄 `# 검증: hermes rolling, 2026-06-10`): RESEARCH.md `curator:` (enabled/interval_hours:168/stale_after_days:30/archive_after_days:90).
   리포트/백업/아카이브 위치를 한 단락으로: `~/.hermes/logs/curator/<timestamp>/REPORT.md`, `~/.hermes/skills/.curator_backups/`, `~/.hermes/skills/.archive/`(복구 가능).

"## 실습: Curator 수동 실행" — RESEARCH.md "Curator CLI" 코드블록(첫 줄 `# 검증: hermes rolling, 2026-06-10`): `hermes curator status` / `hermes curator run` / `hermes curator run --background` / `hermes curator run --dry-run` / `hermes curator rollback`. `hermes curator status` 출력은 LOW이므로 placeholder 코드블록 `# [로컬 실행 후 캡처 필요 — curator status 출력]` + `> 검증 필요:` callout.
