2. "## 개요" — 크론 스케줄러로 무인 반복/지연/특정 시각 작업을 등록한다. 각 작업은 **완전히 격리된 새 AIAgent 세션**에서 실행되며, 결과는 다양한 플랫폼으로 자동 배달된다.

3. "## 개념: 명령은 hermes cron 입니다" — **정확성 핵심:** 크론 CLI는 `hermes cron`이다(NOT `hermes gateway`). `hermes gateway`는 스케줄 실행(60초마다 tick)과 결과 배달을 담당하는 **서비스**로, 크론 작업을 등록/관리하는 명령이 아니다. 아키텍처 한 단락: 스케줄러는 gateway 프로세스(`gateway/run.py`) 안에 살지만 `hermes cron` CLI는 gateway 실행 여부와 무관하게 동작; 작업은 `~/.hermes/cron/`에 JSON 저장; 출력은 `~/.hermes/cron/output/<job_id>/<timestamp>.md`.

4. "## 개념: cron CLI" — RESEARCH.md "Exact CLI Commands" 코드블록(첫 줄 `# 검증: hermes rolling, 2026-06-10`): `hermes cron list/create/edit/pause/resume/run/remove/status/tick`. 별도로 게이트웨이 서비스 명령 코드블록(첫 줄 `# 검증: ...`): `hermes gateway install` / `sudo hermes gateway install --system` / `hermes gateway` — 결과 배달/스케줄 실행에 필요한 서비스임을 명시(크론 명령이 아님).

5. "## 개념: 스케줄 형식" — RESEARCH.md 표(상대 지연 `30m`/`2h`/`1d` = 1회, 인터벌 `every 30m` = 반복, cron 표현식 `0 9 * * *` = 반복, ISO 타임스탬프 = 1회 지정 시각).

6. "## 실습: 예약 작업 등록" — RESEARCH.md "Creating a Job" 예시 코드블록(첫 줄 `# 검증: hermes rolling, 2026-06-10`): 매일 09시 요약(`--deliver telegram`), 2시간마다 블로그 모니터(`--skill blogwatcher --deliver telegram --name "blog-monitor"`), workdir 지정 PR 감사(`--workdir /절대/경로 --deliver "discord:#engineering"`). 등록 후 확인: `hermes cron list`.

7. "## 실습: 즉시 실행과 상태 확인" — 코드블록(첫 줄 `# 검증: ...`): `hermes cron run <name>`(즉시 수동 트리거), `hermes cron status`. `hermes cron run`이 gateway 없이 독립 실행 가능한지는 미검증:
   ```
   > 검증 필요: `hermes cron run <name>`이 gateway 프로세스 없이 독립적으로 실행 가능한지
   > 로컬에서 확인하십시오(gateway 중지 상태로 테스트).
   ```

8. "## 개념: 스킬 주입과 다중 플랫폼 배달" — RESEARCH.md "Skill Injection"(스킬은 프롬프트 실행 **전** 격리 세션에 주입; 단일 `--skill x`, 복수 `--skill x --skill y` 순서대로 로드) 코드블록(첫 줄 `# 검증: ...`). "Multi-Platform Delivery" 표(origin/local/telegram/discord:#channel/복수 "telegram,discord"/all/복합 "origin,all"). 응답은 자동 배달되어 프롬프트에 `send_message` 불필요.

"## 심화: no-agent 모드와 config" — RESEARCH.md "No-Agent Mode" 코드블록(첫 줄 `# 검증: ...`): `--no-agent --script memory-watchdog.sh`(빈 stdout→조용히 종료, 비0 exit→에러 알림, 마지막 줄 `{"wakeAgent": false}`→조용히 종료, 토큰 소비 없음). config.yaml `cron:` 코드블록(첫 줄 `# 검증: ...`): wrap_response/script_timeout_seconds. context_from로 작업 체이닝(에이전트 내부 cronjob 툴)은 한 단락 언급.
