2. "## 개요" — 게이트웨이를 통해 Hermes를 메시징 봇으로 노출하는 것이 이 챕터의 목표. 단일 `hermes gateway` 프로세스 하나가 **24개 플랫폼 어댑터를 동시에** 운영함(RESEARCH "Gateway Architecture"의 ASCII 트리 사용 — 순수 ASCII 다이어그램이므로 주석 생략 가능). "22+"가 아니라 정확히 24개로 기술.

3. "## 개념: 게이트웨이 실행 방식 3가지" — RESEARCH "세 가지 실행 방식" 정확히:
   - `hermes gateway` — 포그라운드 실행 (Ctrl-C로 중지), 개발/테스트용
   - `hermes gateway run` — Docker 컨테이너 내 메인 프로세스로 실행
   - `hermes gateway start` — **설치된 서비스**를 백그라운드로 시작 (먼저 `hermes gateway install` 필요)
   **중요:** "포그라운드로 처음 띄울 때는 `hermes gateway`(또는 `hermes gateway run`)를 쓴다. `hermes gateway start`는 install 이후 서비스 시작용이다"를 명확히 구분. start를 포그라운드 시작 명령으로 잘못 쓰지 말 것.
