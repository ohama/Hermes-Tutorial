# 첫 실행

> **이 챕터를 시작하기 전에**
> - Ch.1 설치 완료 및 `which hermes` 인식 확인
> - 모델/공급자 미설정 상태라면 Ch.3 모델 설정을 먼저 진행하는 것을 권장합니다 (아래 "흔한 오류" 참조)

## hermes 실행하기

Hermes는 두 가지 실행 모드를 제공합니다.

**클래식 CLI 모드** — 플래그 없이 실행합니다. prompt_toolkit 기반 TUI로 시작됩니다.

```bash
# 검증: hermes rolling, 2026-06-09
hermes
```

**모던 TUI 모드** — `--tui` 플래그를 추가합니다. 모달 오버레이, 마우스 선택, 논블로킹 입력을 지원합니다. 공식 문서는 신규 사용자에게 TUI 모드를 권장합니다.

```bash
# 검증: hermes rolling, 2026-06-09
hermes --tui
```

두 인터페이스는 세션·슬래시 커맨드·모든 기능을 공유합니다. 어느 쪽을 선택해도 기능 차이는 없습니다.

## 환영 배너

실행 직후 환영 배너가 표시됩니다. 배너에는 현재 모델명, 작업 디렉터리, 사용 가능한 도구 수, 설치된 스킬 수가 나타납니다. 화면 하단 상태표시줄에는 모델, 토큰 사용량(현재/최대), 컨텍스트 채움 %(색상), 예상 비용, 압축 횟수, 백그라운드 작업 수, 경과 시간이 표시됩니다.

배너의 정확한 모습은 설치된 모델과 환경에 따라 달라지므로 아래는 구조 예시입니다.

```text
# [로컬 실행 후 캡처 필요 — 실제 배너는 모델과 설치 환경에 따라 다름]
╔══════════════════════════════════════╗
║  Hermes Agent  │  <모델명>           ║
║  📁 ~/         │  tools: N  skills: M ║
╚══════════════════════════════════════╝

[상태표시줄: model | N/M tokens | X% | $0.00 | 0 bg | 0s]

> (입력 프롬프트)
```

> 검증 필요: 실제 배너는 로컬 실행 후 스크린샷으로 교체해야 합니다.

## 첫 대화

배너가 나타나면 바로 대화를 시작할 수 있습니다.

1. `hermes` 또는 `hermes --tui` 실행
2. 환영 배너가 나타날 때까지 대기
3. 테스트 메시지를 입력하고 Enter 키를 누릅니다

한국어 예시:

```text
이 레포의 주요 파일을 5줄로 요약해줘.
```

영어 예시 (어떤 모델에서도 동작):

```text
What files are in my current directory?
```

4. 에이전트가 응답합니다. 필요에 따라 터미널 `ls`, 파일 읽기 등 도구가 활성화됩니다.
5. 한 턴 이상 대화가 정상적으로 이어지면 멀티턴 동작이 확인된 것입니다.

에이전트의 실제 응답 출력은 환경에 따라 다릅니다:

```text
# [로컬 실행 후 캡처 필요 — 에이전트 응답은 모델과 작업 디렉터리에 따라 다름]
```

## 첫 대화 성공 기준

공식 퀵스타트에서 정의하는 성공 기준은 다음과 같습니다:

- 배너에 선택한 모델/공급자가 표시된다
- Hermes가 오류 없이 응답한다
- 필요에 따라 도구(터미널, 파일 읽기 등)를 사용한다
- 한 턴 이상 대화가 정상적으로 지속된다

## 세션 재개

이전 대화를 이어서 시작하려면 `--continue` 플래그를 사용합니다.

```bash
# 검증: hermes rolling, 2026-06-09
# 가장 최근 세션 재개
hermes --continue
```

## 주요 CLI 명령어

다음은 자주 사용하는 최상위 명령어 일람입니다. (출처: 공식 CLI 레퍼런스)

| 명령어 | 설명 |
|--------|------|
| `hermes` | 기본 CLI 모드로 시작 |
| `hermes --tui` | 모던 TUI 모드로 시작 |
| `hermes --continue` | 가장 최근 세션 재개 |
| `hermes chat -q "질문"` | 비대화형 단일 질의 |
| `hermes model` | 공급자·모델 설정 마법사 |
| `hermes setup` | 전체 설정 마법사 |
| `hermes setup --portal` | Nous Portal OAuth 원클릭 설정 |
| `hermes tools` | 도구 설정 (대화형) |
| `hermes tools --summary` | 현재 도구 상태 출력 |
| `hermes gateway` | 메시징 플랫폼 서비스 관리 |
| `hermes doctor` | 공급망 CVE 검사 (설정 검증은 `hermes config check`) |
| `hermes config check` | 설정 파일 유효성 검사 |
| `hermes version` | 설치 확인 / 버전 정보 출력 |
| `hermes config show` | 현재 설정 전체 출력 |
| `hermes sessions list` | 세션 목록 |

도움말을 보려면 전역 플래그 `--help`(또는 `-h`)를 사용합니다.

```bash
# 검증: hermes rolling, 2026-06-09
hermes --help
```

> 참고: `hermes help`라는 단독 명령은 없습니다. 도움말은 전역 플래그 `hermes --help` (또는 `-h`)를 사용합니다.

## 세션 내 슬래시 커맨드

대화 중에 슬래시 커맨드로 다양한 기능을 즉시 제어할 수 있습니다. 대소문자를 구분하지 않습니다 (`/HELP` = `/help`).

| 슬래시 커맨드 | 설명 |
|--------------|------|
| `/help` | 사용 가능한 커맨드 목록 |
| `/model` | 현재 모델 표시 / 전환 (설정된 공급자 간) |
| `/tools` | 도구 목록 |
| `/skills browse` | 스킬 허브 탐색 |
| `/voice on` | 음성 모드 활성화 |
| `/reasoning high` | 추론 수준 조정 |
| `/title` | 세션 이름 지정 |
| `/status` | 세션 정보 요약 |
| `/compress` | 수동 컨텍스트 압축 |
| `/rollback` | 마지막 체크포인트로 되돌리기 |

## 키바인딩

| 키 | 동작 |
|----|------|
| `Enter` | 메시지 전송 |
| `Alt+Enter` / `Ctrl+J` / `Shift+Enter` | 줄바꿈 (다음 줄) |
| `Ctrl+G` | 외부 에디터에서 입력 편집 |
| `Ctrl+C` | 에이전트 중단 (두 번 연속 → 강제 종료) |
| `Ctrl+D` | 종료 |

## 흔한 오류 / 주의

> 주의: hermes를 실행했을 때 모델/공급자가 설정되어 있지 않은 경우 "No provider configured" 류 오류가 나타날 수 있습니다. 이 경우 `Ctrl+D`로 종료하고 먼저 Ch.3 모델 설정을 완료하십시오.
