# 소개: Hermes란 무엇인가

Hermes는 Nous Research가 만든 범용 자율 에이전트입니다. IDE 코파일럿이나 챗봇이 아니라, 시간이 지나며 "함께 성장하는" 장기 실행 에이전트입니다. 커맨드라인에서 동작하며 40개 이상의 모델 공급자에 연결됩니다.

> 참고: Hermes UI는 부분 한국어를 지원합니다 (`display.language: ko`). 승인 프롬프트 등 일부 정적 메시지만 번역되며, 에이전트 응답·로그·도구 출력은 영어입니다. 이 튜토리얼은 영어 UI를 기준으로 설명합니다.

## 핵심 개념 1: 멀티모델, 노 락인

Hermes는 Nous Portal, OpenRouter, Anthropic, OpenAI, Gemini, DeepSeek, Ollama 등 40개 이상의 공급자로 라우팅할 수 있습니다. 공급자 전환은 `hermes model` 한 줄로 가능하며 코드 변경이 필요 없습니다. 그래서 모델 설정(Ch.3)이 부수적인 주제가 아니라 일급 관심사입니다.

## 핵심 개념 2: 학습 루프 (Do → Learn → Improve)

이것이 Hermes의 대표 차별점입니다. 단순 반응형 챗봇이 아닙니다.

- **Do(실행):** 내장 도구(터미널, 웹, 파일 검색 등)로 작업을 수행합니다.
- **Learn(학습):** 복잡한 작업(5회 이상의 도구 호출, 또는 새로운 해법 발견) 완료 후 Hermes가 **스킬**(재사용 가능한 문서화된 절차)을 자동 생성해 `~/.hermes/skills/`에 저장합니다.
- **Improve(개선):** 이후 세션에서 시작 시 관련 스킬을 로드합니다. 익숙한 문제에서 점점 빠르고 정확해지며, 스킬은 변형을 만나며 스스로 개선됩니다.

이 루프를 이해하면 왜 컨텍스트 파일·메모리·크론·서브에이전트(후속 챕터)가 모두 연결되는지 알 수 있습니다.

## 그 밖의 특징

- **멀티 플랫폼 접근:** 같은 에이전트를 CLI, Telegram, Discord, Slack, WhatsApp, Signal로 사용할 수 있습니다 (Phase 5에서 다룸).
- **유연한 배포:** $5 VPS나 서버리스(Modal/Daytona)에 배포 가능하며, 노트북에 묶이지 않습니다.
- **개방형 표준:** agentskills.io 오픈 표준과 호환되어 스킬 이식성을 제공합니다.
